提交代码

在本地开发环境中提交代码，这时可以查看服务端仓库是否更新。
出现错误可以查看日志进行调试。

常见错误

1.日志中 output 返回 null
hook.php 接收到了 WebHooks 通知，但是代码拉取不到。
查看日志发现 output 返回 null 。
出现这种情况一般是 服务端 php 禁用了 proc_open 函数。
找到php.ini 搜索 proc_open，删除即可。
2.unable to unlink xxxx : Permission denied
文件权限错误，需要将文件权限修改

注释：修改仓库目录（站点根目录）可读可写
3.error: Your local changes to the following files would be overwritten by merge
由于修改了文件权限后，GIT认为文件已经更改。
所以当本地再次修改代码提交后，WebHooks 会通知服务端GIT PULL，这时代码就冲突了。
